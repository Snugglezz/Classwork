Week 12 Vocab

Unit Test

the process where you test the smallest functional unit of code.

Smoke Test

a preliminary examination to determine the basic functionality of a software, website, or application.

Functional Testing

verifies a program by checking it against design documention or specification

Test Driven Development

a development technique where you must first write a test that fails before you write new functional code.

Non-Functional Testing

evaluates the quality attributes of software systems, emphasizing how a solution performs and operates in various environments rather than its functional capabilities.

Load Test

sends simulated traffic (typically HTTP) to a web server and measures its capability to process the traffic and how.

Regression Test

Ensures that new code changes haven't introduced new bugs in existing functionality

Acceptance Testing

a common type of testing that is probably applied in every project.

code framework

a collection of pre-written code and rules that simplifies and speeds up software development.

Front-end JavaScript frameworks

sets of pre-written code that provide developers with a scalable and maintainable structure for creating user interfaces more efficiently.

Content Management Systems

a web application or software that manages digital content, allowing multiple contributors to create, edit and publish from a single dashboard.

Full-stack web frameworks

Full stack development is the end-to-end process of building a web application, covering everything from front-end design to back-end functionality.

header, main, footer tags

Defines a footer for a document or section.
